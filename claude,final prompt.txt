# Cooking Lab — Complete Project Summary

## Project Overview

**Cooking Lab** is an AI-powered recipe generation web application built as an 8th semester Computer Engineering project by **Bhavik Ratanpara**. The app allows users to generate accurate recipes with exact quantities, save them, share with community, and explore recipes shared by others — all powered by Google Gemini AI.

## Firestore Database Structure

```
users/
  {userId}/
    savedRecipes/
      {recipeId}/
        recipeName: string
        cuisine: string
        servings: number
        dietType: "veg" | "non-veg"
        language: string
        ingredients: array
        steps: array
        generatedAt: timestamp
        isFavourite: boolean
        isPublic: boolean
        savedFromExplore: boolean
        originalSharedBy: string
        originalSharedByName: string
        originalRecipeId: string
        likes: number
        likedBy: array

publicRecipes/
  {recipeId}/
    (same fields as savedRecipes)
    sharedBy: string (userId)
    sharedByName: string
    sharedAt: timestamp
    likes: number
    likedBy: array (userIds)
```

## Firestore Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null 
        && request.auth.uid == userId;
    }
    match /publicRecipes/{recipeId} {
      allow read: if true;
      allow create: if request.auth != null;
      allow update: if request.auth != null;
      allow delete: if request.auth != null
        && resource.data.sharedBy == request.auth.uid;
    }
  }
}
```

## Authentication Flow

```
Methods supported:
1. Email + Password
2. Google Sign In (signInWithPopup)

Firebase Console setting:
"Link accounts that use same email" — ON

Email verification: NOT implemented (removed)
Password reset: NOT implemented (removed)

Account conflict handling:
If email+pass user tries Google login
→ Error: auth/account-exists-with-different-credential
→ User prompted for password
→ Accounts linked via linkWithCredential
→ Both methods work after linking

Warning shown on login page:
"If registered with email+pass, use that to login"
"Switching to Google changes your login method"
```

## Core Features

### 1. AI Recipe Generation
```
User fills form → Gemini API called →
Complete recipe generated with:
  - Exact ingredient quantities
  - Step by step instructions
  - Scaled to user's servings
  - In user's chosen language
  - Veg/Non-veg specific
```

### 2. Save Recipes
```
Generated recipe → Save button →
Stored in Firestore under users/{uid}/savedRecipes
Accessible from My Recipes page
```

### 3. Favourites
```
Heart icon on recipe cards →
isFavourite: true in Firestore
Filter by favourites in My Recipes
```

### 4. Share to Explore (Public)
```
My Recipes card → Share button →
If not public: modal asks to publish first
Recipe copied to publicRecipes collection
isPublic: true set on savedRecipe
sharedBy, sharedByName, sharedAt saved
```

### 5. Save from Explore
```
Explore card → Save button →
Checks if already saved (duplicate prevention)
Saved to user's collection with:
  savedFromExplore: true
  originalRecipeId: (public recipe ID)
  originalSharedBy, originalSharedByName
```

### 6. Like System
```
Heart button on Explore cards
Toggle like/unlike with optimistic UI
likes counter + likedBy array in Firestore
Any logged in user can like any recipe
Sort by Most Liked available
```

### 7. Share Link
```
Share2 icon on all recipe cards
Mobile: native share sheet (navigator.share)
Desktop: copy link to clipboard + "Link Copied!" toast

Logic:
  savedFromExplore → share /explore/recipe/{originalRecipeId}
  Own recipe + isPublic → share /explore/recipe/{id}
  Own recipe + not public → modal to publish first
```

### 8. Search
```
Search bar in navbar
Search within My Recipes page
Search within Explore page
Filters by recipe name
```

### 9. Language Filter (Explore)
```
Dynamic language detection from recipes
Dropdown with available languages
Prevents users from seeing recipes
they can't understand
```

### 10. Recipe Persistence (localStorage)
```
Recipe saved to localStorage after generation
Form values also saved
On return to home — recipe restored
Banner shown with Generate New / Clear options
```

### 11. Pagination
```
Explore: Load More button (12 per load)
My Recipes: Page numbers (12 per page)
Resets on filter change
Scroll to top on page change
```

## Problems Solved

```
1. Recipe quantity scaling
   → AI generates exact quantities for any servings

2. Multi-language recipes
   → Generate in any language user chooses

3. Recipe discovery
   → Community Explore page with filters

4. Duplicate saves from Explore
   → originalRecipeId check prevents duplicates

5. Recipe persistence across navigation
   → localStorage saves last generated recipe

6. Non-English recipes in Explore
   → Language filter allows users to filter by language

7. Like spam/multiple likes
   → likedBy array prevents duplicate likes

8. Auth account conversion
   → Warning text + linkWithCredential merge logic

9. Mobile viewport zoom issue
   → Viewport meta tag + overflow:hidden fixes

10. Explore page performance
    → Pagination with Load More (12 cards at a time)
```

## Key Design Decisions

```
1. Lottie animation instead of static image
   → Lighter than video, more engaging than image

2. Floating food images (Zomato style)
   → Scroll triggered — slides in from sides
   → SVG curved lines behind images
   → Hidden on mobile completely

3. Magic line navbar
   → Smooth sliding underline between nav items
   → No pill buttons — clean text only

4. Warm cream background (#F5F0EB) for light mode
   → More premium than pure white
   → Food-appropriate warmth

5. Optimistic UI for likes
   → Count updates instantly
   → Server syncs in background
   → Reverts if server fails

6. Community Explore public recipes
   → Anyone can view without login
   → Must login to save/like/share

7. No manual recipe database
   → Only AI generated recipes
   → Keeps app focused on AI value
```
